## 1. 系统概述

MindBridge Assist 是一套**在线心理评估与行为能力分析辅助支持系统**，核心定位为**心智障碍群体日常沟通辅助与情绪/需求表达支持**。

### 1.1 业务使命

- **沟通桥梁** — 为心智障碍用户构建多模态（文本、语音、图像、触控）沟通通道
- **行为洞察** — 通过 AI 驱动的行为分析引擎，识别行为模式与情绪状态
- **需求表达** — 帮助难以用传统方式表达的用户准确传达自身需求
- **辅助决策** — 为照护者、治疗师、教育工作者提供数据驱动的决策支持

### 1.2 目标用户

| 用户角色         | 描述                                     |
| ---------------- | ---------------------------------------- |
| 心智障碍用户     | ASD、智力障碍、发育迟缓、认知障碍等群体  |
| 照护者/家属      | 日常照料、沟通协助、情绪监测              |
| 专业治疗师       | 心理治疗师、行为分析师、语言治疗师        |
| 教育工作者       | 特教老师、融合教育支持人员                |
| 系统管理员       | 平台运维、数据管理、合规审计              |

### 1.3 核心价值主张

## 2. 架构原则

| 编号 | 原则                    | 说明                                                     |
| ---- | ----------------------- | -------------------------------------------------------- |
| AP1  | **安全与合规优先**      | HIPAA/GDPR 默认内置，零信任架构，数据最小化原则           |
| AP2  | **可访问性为核心**      | WCAG 2.1 AA 级标准贯穿所有交互层，心智障碍友好设计        |
| AP3  | **微服务自治**          | 每个服务独立部署、独立扩展、独立故障域                     |
| AP4  | **AI 可解释性**         | 所有 AI 决策输出附带可解释性标注，支持人工审核             |
| AP5  | **实时响应**            | 核心交互链路端到端延迟 < 500ms，评估结果 < 3s              |
| AP6  | **弹性与容错**          | 服务降级策略、熔断机制、优雅回退，99.95% SLA               |
| AP7  | **数据隐私保护**        | 端到端加密、匿名化处理、用户数据主权、可追溯审计            |
| AP8  | **多租户隔离**          | 机构级数据隔离，租户级资源配置，跨租户零数据泄露            |

## 4. 分层架构详解

### 4.1 展示层 (Presentation Layer)

| 组件             | 技术栈              | 职责说明                                       |
| ---------------- | ------------------- | ---------------------------------------------- |
| Web 应用         | React 18 + Next.js 15.x | 响应式 Web 界面，服务端渲染，SEO 优化            |
| 移动端           | React Native        | iOS/Android 原生体验，离线缓存，推送通知         |
| 治疗师工作台     | React + D3.js       | 数据可视化仪表盘，个案管理，批量分析             |
| 管理后台         | React + Ant Design  | 系统配置，用户管理，审计日志，租户管理           |
| 无障碍组件库     | 自研 a11y UI Kit    | WCAG 2.1 AA 级组件，认知友好交互模式              |

**无障碍设计要点：**
- 大按钮、高对比度、简化布局
- 语音导航与屏幕阅读器优化
- 触控容错区域 ≥ 44×44pt
- 认知负荷最小化（单任务界面、渐进式引导）

### 4.4 数据层 (Data Layer)

## 13. 成本优化方案

### 13.1 计算资源优化

| 优化策略         | 具体措施                                                   | 预估节省   |
| ---------------- | ---------------------------------------------------------- | ---------- |
| Spot 实例        | 非核心服务(AI训练、批处理报告)使用 Spot 实例，配置优雅终止  | 60-70%     |
| 自动扩缩容       | HPA 基于 CPU/GPU/队列深度自动伸缩，低谷期缩至最小副本       | 30-40%     |
| GPU 共享         | Triton 推理服务器多模型共享 GPU，MIG 实例分割               | 40-50%     |
| 混合实例类型     | 通用计算用 Graviton(ARM)，AI 推理用 GPU，轻量任务用 Burstable | 20-30%     |
| 预留实例         | 核心服务(数据库、网关)购买 1 年预留实例                     | 30-35%     |
| 调度优化         | 非实时任务(报告生成、模型训练)调度至低谷时段执行             | 15-25%     |

### 13.2 存储成本优化

| 优化策略         | 具体措施                                                   | 预估节省   |
| ---------------- | ---------------------------------------------------------- | ---------- |
| 冷热数据分离     | 热数据(SSD)：近 30 天评估/行为数据；冷数据(HDD/S3)：历史归档 | 50-60%     |
| 压缩策略         | PostgreSQL TOAST 压缩、Parquet 列式存储归档、ZSTD 压缩备份  | 40-60%     |
| 生命周期策略     | S3 生命周期：Standard → IA(30天) → Glacier(90天) → 深度归档(365天) | 60-70%     |
| 重复数据删除     | 多媒体文件去重、模型权重共享存储、增量备份                  | 20-30%     |
| 存储分层         | MinIO 热温冷分层、Elasticsearch 冷热节点分离               | 30-40%     |

### 13.3 网络成本优化

| 优化策略         | 具体措施                                                   | 预估节省   |
| ---------------- | ---------------------------------------------------------- | ---------- |

| DNS 优化         | Route53 智能路由，就近接入，减少跨区域延迟与流量            | 10-15%     |
